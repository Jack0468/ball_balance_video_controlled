# FPGA Main Controller

This directory contains the core Verilog modules that synthesize the hardware brain of the robot on the Opal Kelly XEM3010 (Spartan-3). 

This unified architecture handles:
- Capturing real-time video from the OV7670 camera.
- Reading coordinates from the TI ADS1675 touchscreen ADC.
- Hardware Proportional (P) control for 3 stepper motors (TMC2208/A4988).
- Multiplexing all sensor data and sending it to the Host PC via a high-speed Opal Kelly `PipeOut`.

## Architectural Notes
- **Independent PLL Clocking (`clk1`)**: To ensure extreme stability, the motors, ADC, and camera master clock run on an independent 48MHz clock generated by the XEM3010's onboard PLL (`PLL22393`), rather than relying on the USB-dependent `ti_clk`. Python configures this PLL on startup.
- **Hardware PID**: The FPGA autonomously computes the motor step pulses based on a 32-bit target position provided by Python.
- **Active-Low Enable**: The `enable_pin` is internally tied to `0` (LOW) to permanently turn on the physical stepper motor drivers.

## Synthesis Instructions (Xilinx ISE 14.7)

Because the XEM3010 uses a legacy Xilinx Spartan-3 FPGA, you must compile this project using **Xilinx ISE 14.7** (modern Vivado releases do not support the Spartan-3 family).

### 1. Create the Project
1. Open Xilinx ISE 14.7.
2. Select **File > New Project**.
3. Name the project `main_controller` and set the directory to this folder (`fpga/main_controller`).
4. Set the family to **Spartan3** and select the specific device model matching your XEM3010 (e.g., `xc3s1500-4fg320`).

### 2. Add Source Files
1. Right-click the hierarchy window and select **Add Source...**
2. Add all `.v` files in this directory.
3. Ensure `fpga_usb_streamer.v` is set as the **Top-Level Module**.

### 3. Add Opal Kelly Dependencies
You must include the proprietary FrontPanel HDL files provided by Opal Kelly:
1. Locate the `okLibrary.v` file from your Opal Kelly SDK installation and add it to the ISE project.
2. Locate the `okCoreHarness.ngc` (or equivalent netlist) from the SDK and add it to the ISE project.

### 4. Constraints File (UCF)
You must create a User Constraints File (`.ucf`) to map the Verilog ports to physical pins. Ensure you map:
- The Opal Kelly host interface pins (`hi_in`, `hi_out`, `hi_aa`, etc.).
- `clk1` to the physical pin connected to `PLL0_0`.
- The motor pins (`step1_pin`, `dir1_pin`, and `enable_pin`).
- The camera and ADC pins.

### 5. Generate Bitstream
1. In the Processes window, double-click **Generate Programming File**.
2. Wait for synthesis, translation, mapping, and routing to complete. 
3. This will produce an `fpga_usb_streamer.bit` file.

## Testing the Hardware

Once the `.bit` file is compiled, flash it to the board using the Python scripts in the `host_software` directory:

1. **Test the Motors:**
   Run `python host_software/tools/test_motors.py`. This script will configure the PLL clock, zero the hardware step counters, and push a 3200-step target into the holding registers (WireIns `0x01` through `0x06`). It then fires the atomic latch trigger (`0x40`) to smoothly move all three motors simultaneously.
2. **Full Vision Pipeline:**
   Run `python host_software/data_collection/collect_training_data.py`. This fully integrates the PID controllers, the Inverse Kinematics, the I2C camera bit-banging, and the real-time blocking PipeOut (`0xA0`) to generate autonomous ML training data!
